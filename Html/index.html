Difference between UTF-8 and UTF-16

Character-set Description
UTF-8 A character in UTF8 can be from 1 to 4 bytes long. UTF-8 can represent any character in the Unicode standard. UTF-8 is backwards compatible with ASCII. UTF-8 is the preferred encoding for e-mail and web pages
UTF-16 16-bit Unicode Transformation Format is a variable-length character encoding for Unicode, capable of encoding the entire Unicode repertoire. UTF-16 is used in major operating systems and environments, like Microsoft Windows, Java and .NET.
HTML 4 supports UTF-8. HTML 5 supports both UTF-8 and UTF-16!

The Unicode Standard covers (almost) all the characters, punctuations, and symbols in the world. Unicode enables processing, storage, and transport of text independent of platform and language. The default character encoding in HTML-5 is UTF-8. If an HTML5 web page uses a different character set than UTF-8, it should be specified in the meta tag

The Difference Between Unicode and UTF-8

Unicode is a character set. UTF-8 is encoding. Unicode is a list of characters with unique decimal numbers (code points). A = 65, B = 66, C = 67, .... This list of decimal numbers represent the string "hello": 104 101 108 108 111 Encoding is how these numbers are translated into binary numbers to be stored in a computer: UTF-8 encoding will store "hello" like this (binary): 01101000 01100101 01101100 01101100 01101111 Encoding translates numbers into binary. Character sets translates characters to numbers.

Tags introduced in HTML5

<article> Tag

The <article> tag specifies independent, self-contained content. It should make sense on its own, and it should be possible to read it independently from the rest of the web page.

<aside> Tag

The <aside> tag is used to define some content aside from the content it is placed in. It can be used for sidebars, pull quotes, or advertising.

<details> and <summary> Tags

The <details> tag is used to create an interactive widget that the user can open or close. The <summary> tag defines a visible heading for the <details> element.

The <figure> tag represents content like illustrations, diagrams, photos, code listings, etc. The <figcaption> tag is used to define a caption for the <figure> element.

<footer> Tag

The <footer> tag defines a footer for a document or a section. It typically contains authorship information, copyright information, contact information, etc.

<header> Tag

The <header> tag specifies a container for introductory content or a set of navigational links. It can contain some heading elements, a logo, or any other introductory content.

<nav> Tag

The <nav> tag defines a set of navigation links. It is intended for major block of navigation links; however, not all links in a document should be inside a <nav> element.

<section> Tag

The <section> tag defines a standalone section in a document. It does not introduce a new section in the outline of a document, but adds thematic grouping to the content.

<time> Tag

The <time> tag defines a specific period in time or a duration. It can be used to encode dates and times in a machine-readable way.

The <mark> tag is used to highlight parts of text, such as search query words within a document.

<main> Tag

The <main> tag specifies the main content of a document. There should be only one <main> element per page, and it should be unique from content in other elements such as <header>, <footer>, and <nav>.

The <canvas> tag is used to draw graphics on the fly, such as during gameplay or other interactive applications. It requires JavaScript to actually draw the graphics.

The <datalist> tag provides an "autocomplete" feature on <input> elements. Users will see a dropdown of options based on the input.

The <video> tag is used to embed video content. Similarly, there's an <audio> tag for audio content.

The <source> tag, when nested inside <video> or <audio> tags, allows you to provide multiple media files for compatibility with different browsers.

The <track> tag is used with <video> and <audio> to provide captions or subtitles.

The <picture> tag, in conjunction with <source>, allows for responsive image loading, choosing the best image source based on the viewport size or screen resolution.

The <template> tag holds client-side content that you can reuse and instantiate using JavaScript. The content inside a <template> tag is not rendered until activated by JavaScript.
